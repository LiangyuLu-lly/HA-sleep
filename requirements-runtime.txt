# Add-on 镜像运行时依赖。
#
# 设计说明
# ----------
# v1.3.0 起不再加载本地深度学习模型（CNN-BiLSTM 已废弃），
# v1.6.1 起 ``src/`` 和 ``scripts/`` 内也没有任何 ``import
# numpy`` / ``np.`` 的引用（可用 ``grep -R "numpy" src/ scripts/``
# 自查）——numpy 因此被移出运行时依赖，镜像再瘦约 5 MB。
#
# 当前保持最小：aiohttp 用来跑 HA REST + WebSocket 客户端以及
# 内嵌的 Web UI server。
#
# v3.0.0（algorithmic-moat）破例理由
# -----------------------------------
# 为支撑 4 个算法护城河中的 3 个本地推理模块，v3.0.0 重新引入
# 重型科学计算依赖。每个新依赖必须有明确归属模块；CI 静态扫描
# 会校验其在 ``src/`` 内有 ``import`` 路径覆盖（防止「装了不用」
# 造成无谓体积膨胀，详见 R12.4 / P11）。版本范围全部固定大版本，
# 避免破坏性升级（R12.1）。
#
# 镜像体积基线随之从 ~15 MB 提升到 ~80 MB（见
# ``.github/baseline_image_size.txt``），CI 守护上限 96 MB（基线
# ×1.20）；超出即构建失败（R12.3 / PR4）。
#
# 必需的不变量：在只装下面列出的依赖的前提下，``import src``
# 必须成功——orchestrator 的冷启动路径不允许触发
# ``ModuleNotFoundError``。
aiohttp>=3.9.0

# v3.0.0 新增 —— 算法护城河本地推理依赖
# numpy:       BAO（GP 后验向量化运算）+ CAE（bootstrap 95% CI）
numpy>=1.24,<2.0
# scipy:       BAO（``scipy.linalg.cho_factor`` / ``cho_solve`` 解
#              cholesky 分解后的线性系统，实现 RBF kernel GP 后验）
scipy>=1.10,<2.0
# onnxruntime: EMST（端侧 INT8 量化 stage_predictor.onnx 推理）
# v3.0.0+ Debian glibc base：onnxruntime 直接装 manylinux wheel 即可。
onnxruntime>=1.16,<2.0
